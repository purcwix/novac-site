Introduction

Novac is a scripting language built from scratch: its own lexer, parser, and bytecode virtual machine, written in C++. It's designed for scripts that need to reach outside the sandbox — calling into native C libraries through libffi, or driving Windows COM objects — without giving up a REPL-first, scripting language feel.

This section is a living document. It grows alongside the language itself, so expect gaps — if you hit one, the source and stdlib are the ground truth in the meantime.

Building from source

Novac requires CMake 3.20+ and a C++20 compiler (GCC 13+, Clang 16+, or MSVC 2022+).

git clone https://github.com/purcwix/novac.git
cd novac
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release -j4

This produces two binaries: novac, the interpreter/VM, and nv+, the package manager.

Using the REPL

Running novac with no arguments drops you into the REPL, with raw-mode line editing and arrow-key history navigation built in.

$ novac
> let x = 21
> x * 2
42

Built-in REPL commands:

  • .edit — open the current buffer in your $EDITOR
  • .time — time the execution of the next expression
  • .history — show REPL input history
  • .save <file> — save the current session to a file

Basic syntax

let name = "world"
print("hello, " + name)

fn add(a, b) {
    return a + b
}

let nums = [3, 1, 4, 1, 5, 9]
print(nums.sort().reverse())

Standard library

A growing set of built-in modules, including:

  • Std.StringScanner / Std.StringBuilder — text processing
  • Std.Lexer — build your own tokenizers on top of Novac's own
  • Std.LLM — a llama.cpp wrapper with KV-cache diffing
  • Std.Getch / Std.Kbhit / Std.Putch — cross-platform terminal I/O

FFI & libffi

Novac can call into C libraries directly, including native callbacks via ffi_closure, variadic calls, and struct/union marshalling by value.

let libm = Std.Ffi.load("libm.so.6")
let sqrt = libm.bind("sqrt", "double", ["double"])
print(sqrt(2))

Windows COM

On Windows, Novac scripts can drive COM objects directly — including embedding WebView2 — without hand-writing interface bindings.

nv+ package manager

nv+ installs and publishes Novac packages through a GitHub PR–based registry workflow (nova-registry) — no central package server required. 

nv+ install nvo-semver
nv+ install nvo-cron
