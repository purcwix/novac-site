Introduction

Novac is a scripting language built from scratch — its own lexer, parser, and C++ executor, with no third-party runtime underneath. It's designed to feel like a modern scripting language while still being able to reach into native territory: loading shared libraries, calling C functions directly, driving Win32 GUIs, and running worker threads.

This documentation is a living reference. The examples/ directory in the repo is the most complete and authoritative source of runnable Novac code.

Building from source

Requires CMake 3.20+ and a C++20 compiler (GCC 13+, Clang 16+, or MinGW).

git clone https://github.com/purcwix/novac.git
cd novac
cmake -B build -DCMAKE_BUILD_TYPE=Release
cmake --build build --config Release -j4

This produces two binaries: novac (the interpreter) and nv+ (the package manager).

Using the REPL

Running novac with no arguments starts the REPL with raw-mode line editing and arrow-key history.

$ novac
Type .exit to exit, .help for commands

> let x = 21;
> x * 2;
42

Built-in REPL commands:

  • .edit — enter multi-line edit mode; .editend to run, .editcancel to abort
  • .time — time the next expression
  • .history — print input history
  • .save — save session to a file
  • .help — list all commands

Variables

let, const, and var all declare variables. Destructuring works on both objects and arrays.

let x = 10;
const PI = 3.14159;
var mutable = 42;
mutable = 100;

// destructuring
let { a, b } = { a: 1, b: 2 };
let [first, second] = [10, 20];
let [head, ...tail] = [1, 2, 3, 4];

Novac also has smart variable modifiers:

let x frozen = 42;                          // cannot be reassigned
let score tracked = 0;                      // prints on every change
let name nonull = "Alice";                  // throws if set to null
let id once = 1001;                         // can only be assigned once
let heavy lazy = (() => expensiveOp())();   // computed on first access
let celsius = 100;
let f getter (v => v * 9/5 + 32) = celsius; // transform on read
let upper setter (v => v.toUpperCase()) = "alice"; // transform on write

Functions

// declaration
function greet(name) {
  return f"Hello, {name}!";
}

// default params
function power(base, exp = 2) {
  return base ** exp;
}

// rest params
function sum(...nums) {
  let total = 0;
  each n of nums { total += n; }
  return total;
}

// arrow functions
let double = x => x * 2;
let add = (a, b) => a + b;

// 'give' is an alias for return
function triple(n) {
  give n * 3;
}

// closures
function makeCounter() {
  let count = 0;
  return () => { count++; return count; };
}
let counter = makeCounter();
Std.println(counter()); // 1
Std.println(counter()); // 2

Control flow

// if / else if / else
if x > 20 {
  Std.println("big");
} else if x > 10 {
  Std.println("medium");
} else {
  Std.println("small");
}

// unless (runs when condition is FALSE)
unless x == 0 { Std.println("x is not zero"); }

// ternary
let label = x > 10 ? "over ten" : "ten or less";

// postfix if
Std.println("positive") if x > 0;

// switch
switch day {
  case "Mon": Std.println("Monday"); break
  case "Fri": Std.println("Friday"); break
  default:    Std.println("Other day")
}

Loops

// C-style for
for let i = 0; i < 5; i++ { Std.println(i); }

// for-of
for let v of [10, 20, 30] { Std.println(v); }

// for-in (object keys)
for let k in { a: 1, b: 2 } { Std.println(k); }

// each — Nova-style for-of with optional index
each v, idx of arr { Std.println(f"arr[{idx}] = {v}"); }

// while / until / do-while
while i < 5 { i++; }
until i >= 5 { i++; }    // runs while condition is FALSE
do { i++; } while i < 5;

// repeat N times
repeat 4 { Std.println("repeat!"); }

// ranges (inclusive)
for let n of 0..5 { Std.println(n); }

Classes

class Animal {
  name: ""
  sound: ""

  constructor(name, sound) {
    this.name = name;
    this.sound = sound;
  }

  speak() {
    Std.println(f"{this.name} says {this.sound}");
  }
}

class Dog extends Animal {
  breed: ""

  constructor(name, breed) {
    Super(name, "woof");
    this.breed = breed;
  }
}

let dog = Dog("Rex", "Labrador");
dog.speak();
Std.println(dog instanceof Dog);    // true
Std.println(dog instanceof Animal); // true

Structs & enums

struct Point {
  x: number = 0,
  y: number = 0
}
let p = Point({ x: 3, y: 4 });
Std.println(p.x, p.y);

enum Direction { North, South, East, West }
let dir = Direction.North;
Std.println(dir.variant);   // "North"
Std.println(dir.enumType);  // "Direction"

Pattern matching

function classify(x) {
  match x {
    when 0         { return "zero"  }
    when 1, 2, 3   { return "small" }
    default        { return "other" }
  }
}

// match on type
function describe(val) {
  match Std.typeOf(val) {
    when "number" { return f"number: {val}"   }
    when "string" { return f"string: \"{val}\"" }
    default       { return f"other"           }
  }
}

Async & generators

// async / await
function async fetchData() {
  await Std.sleep(10);
  return { status: 200, data: "hello" };
}
let result = await fetchData();
Std.println(result.status);

// generators
function generator count(start, end) {
  for let i = start; i <= end; i++ {
    yield i;
  }
}
let gen = count(1, 5);
Std.println(gen.next()); // 1
Std.println(gen.next()); // 2

Scope variables

Every scope in Novac exposes three read-only introspection variables, similar to globalThis in JavaScript:

  • localScope — an object snapshot of all variables in the current scope
  • parentScope — an object snapshot of the immediately enclosing scope's variables
  • globalScope — an object snapshot of the top-level global scope (all built-ins and top-level declarations)
let x = 10;
function demo() {
  let y = 20;
  Std.println(localScope.y);        // 20
  Std.println(parentScope.x);       // 10
  Std.println(globalScope.Std);     // [object]
}
demo();

All three are read-only — assigning to them throws a runtime error.

Std.* overview

Novac's standard library lives under the Std namespace. A selection of available modules:

  • Std.println / Std.print / Std.input — basic I/O
  • Std.typeOf / Std.typeof — type introspection
  • Std.Math — math utilities (.sqrt, .abs, .floor, .random, .PI, …)
  • Std.String — string utilities (.fromCharCode, …)
  • Std.Object — .keys, .values, .entries
  • Std.FS — filesystem: .Exists, .IsFile, .IsDirectory
  • Std.Random — .shuffle and more
  • Std.Regex — regular expressions
  • Std.StringScanner / Std.StringBuilder — text processing
  • Std.Lexer — build tokenizers on top of Novac's own lexer
  • Std.Fiber — cooperative fibers / coroutines
  • Std.WorkerThread — run Novac source strings in parallel worker threads
  • Std.Getch / Std.Kbhit / Std.Putch — cross-platform raw terminal I/O
  • Std.Timers — timers and scheduling
  • Std.Processes — process management
  • Std.Dlopen / Std.Ccall — native library loading and calling
  • Std.Windows.GUI — Win32 window and control creation (Windows only)
  • Std.Satisfies — runtime interface conformance check
  • Std.SetOverload / Std.SetOverloadRaw — operator overloading
  • Std.eval / Std.include — dynamic code evaluation and file inclusion

nv+ package manager

nv+ installs and publishes packages through a GitHub PR–based registry — no central server. 

nv+ install nvo-semver
nv+ install nvo-cron
nv+ publish        // creates repo + opens PR to nova-registry
nv+ list
nv+ search <name>

Packages export symbols using export { name: value } and are imported with from "path" import Name or Std.include(name).
